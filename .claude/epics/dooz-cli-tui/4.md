# Task: Fragment trait and registry

## Description

Implement the modular fragment system: Fragment trait defining the interface, FragmentId enum, FragmentRegistry for registration, and fragment routing in the main loop.

## Acceptance Criteria

- [x] Fragment trait with `Model`, `update()`, `view()` methods
- [x] FragmentId enum with Chat variant (for Fragment1)

## Technical Details

### Fragment Trait (src/fragments/fragment.rs)
```rust
use ratatui::Frame;
use crate::action::Action;

pub trait Fragment {
    type Model: Default;
    
    fn update(&mut self, model: &mut Self::Model, action: Action) -> Option<Action>;
    fn view(&self, model: &Self::Model, frame: &mut Frame);
}
```

### FragmentId (src/fragments/mod.rs)
```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash)]
pub enum FragmentId {
    Chat,  // Fragment1
    // Future: Settings,
    // Future: Help,
}
```

### Fragment Registry Pattern
```rust
pub struct FragmentRegistry<F: Fragment> {
    fragments: HashMap<FragmentId, F>,
    active: FragmentId,
}

impl<F: Fragment> FragmentRegistry<F> {
    pub fn new() -> Self;
    pub fn register(&mut self, id: FragmentId, fragment: F);
    pub fn set_active(&mut self, id: FragmentId);
}
```

### Routing in App
```rust
impl App {
    fn update(&mut self, action: Action) -> Option<Action> {
        self.registry.update(&mut self.registry.active, action)
    }
    
    fn view(&self, frame: &mut Frame) {
        self.registry.view(frame);
    }
}
```

# Task: Session storage layer

## Description

Implement session persistence layer: data types (Session, Message), store operations (create, read, update, list), and `.doz/sessions/` directory management.

## Acceptance Criteria

- [ ] Session struct with id, title, created_at, updated_at, messages
- [ ] Message struct with id, role (user/assistant), content, timestamp
- [ ] Session store can create new sessions
- [ ] Session store can load existing session by ID
- [ ] Session store can save/update session
- [ ] Session store can list all sessions
- [ ] Auto-create `.doz/sessions/` directory on first run
- [ ] Unit tests for session store

## Technical Details

### Types (src/session/types.rs)
```rust
use chrono::{DateTime, Utc};
use serde::{Deserialize, Serialize};
use uuid::Uuid;

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct Session {
    pub id: Uuid,
    pub title: String,
    pub created_at: DateTime<Utc>,
    pub updated_at: DateTime<Utc>,
    pub messages: Vec<Message>,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct Message {
    pub id: Uuid,
    pub role: MessageRole,
    pub content: String,
    pub timestamp: DateTime<Utc>,
}

#[derive(Debug, Clone, Serialize, Deserialize, PartialEq)]
pub enum MessageRole {
    User,
    Assistant,
}
```

### Store (src/session/store.rs)
```rust
use std::path::PathBuf;
use uuid::Uuid;

pub struct SessionStore {
    sessions_dir: PathBuf,
}

impl SessionStore {
    pub fn new() -> Result<Self>;
    pub fn create_session(&self, title: String) -> Result<Session>;
    pub fn get_session(&self, id: Uuid) -> Result<Option<Session>>;
    pub fn save_session(&self, session: &Session) -> Result<()>;
    pub fn list_sessions(&self) -> Result<Vec<Session>>;
}
```
